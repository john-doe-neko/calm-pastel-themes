Это расширения для фаерфокс, оно написано с ориентированностью на простые или средние по сложности сайты и считает своей целью изменить их внешний вид.
1. Идея и цель проекта
·	Создать простое, красивое и полезное расширение для браузера Firefox
·	Дать пользователям возможность мгновенно менять настроение всей веб-среды
·	Использовать исключительно пастельные, мягкие, уютные цветовые палитры
·	Сделать переключение тем максимально удобным и приятным
·	Главная цель — чтобы человек, открыв браузер, сразу почувствовал себя спокойнее и уютнее
2. Основные технические решения
·	Manifest Version 3 (современный стандарт расширений)
·	Совместимость конкретно с Firefox (учтены его особенности в 2025–2026 гг.)
·	Переключение между 5 тщательно подобранными пастельными темами:
1.	Lavender Bliss (лавандовый рай)
2.	Fresh Mint (свежая мята)
3.	Soft Peach (мягкий персик)
4.	Sakura Dream (цветущая сакура)
5.	Bluebell Fog (туман колокольчиков)
·	Использование CSS-переменных и переопределение встроенных переменных сайтов
·	Программная инъекция стилей через browser.scripting.insertCSS/removeCSS
·	Сохранение выбранной темы через browser.storage.local
·	Красивый минималистичный попап с крупными кнопками-«настроениями»
3. Основные этапы разработки (хронология)
1.	Исследование современных ограничений Manifest V3 в Firefox 
2.	Выбор между service_worker и классическим background scripts → остановка на scripts 
3.	Подбор и создание 5 гармоничных пастельных палитр (много ручного подбора оттенков) 
4.	Разработка базовой структуры расширения (manifest + popup + content + background) 
5.	Реализация механизма динамической смены тем без перезагрузки страницы 
6.	Тестирование на самых популярных сайтах (YouTube, GitHub, Wikipedia, Reddit, Hacker News) 
7.	Обнаружение и анализ проблемы с Pinterest → вывод о необходимости специализированных инструментов 
8.	Глубокая адаптация тем именно под GitHub (переопределение ~20 официальных CSS-переменных) 
9.	Улучшение UX попапа: визуальная индикация активной темы, крупные названия настроений 
10.	Финальное тестирование стабильности на реальных сценариях использования (январь 2026)
4. Полученные технические навыки и знания
·	Глубокое понимание различий Manifest V2  V3
·	Работа с browser.scripting API (insertCSS, removeCSS, origin: "USER")
·	Особенности реализации background в Firefox (почему service_worker пока отключён)
·	Работа с Content Security Policy сайтов и способы его обхода/учёта
·	Переопределение встроенных CSS-переменных крупных сайтов (особенно GitHub)
·	Создание приятного UX для расширений (popup-дизайн, мгновенная обратная связь)
·	Понимание, какие сайты «дружелюбны» к глобальным темам, а какие нет
·	Навыки отладки расширений через about:debugging
5. Достижения и результаты
·	Полностью рабочее расширение с пятью темами
·	Стабильная смена тем без перезагрузки страниц на большинстве сайтов
·	Отличная работа на GitHub (включая страницы /topics/*)
·	Очень приятный и спокойный визуальный опыт на светлых темах
·	Осознанное понимание ограничений и сильных сторон глобальных тем в 2026 году
·	Готовый продукт, который реально хочется использовать каждый день
6. Выводы и планы на будущее
·	Глобальные темы — это красиво, но технически сложная задача в современном вебе
·	Самые большие проблемы создают сайты с Shadow DOM и строгим CSP (Pinterest, Instagram, Discord и т.п.)
·	Для максимального комфорта в будущем планируется:
·	интеграция с популярными userstyle-менеджерами
·	создание тем, специфичных под отдельные крупные сайты
·	добавление поддержки тёмных пастельных вариантов
·	возможно — плавные переходы между темами (transition на :root)
·	публикация в addons.mozilla.org после доработки иконок и описания
